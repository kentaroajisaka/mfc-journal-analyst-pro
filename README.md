# mfc-journal-analyst-pro

マネーフォワードクラウド会計（MFクラウド会計）の仕訳データを分析し、**記帳代行の引き継ぎシート**と**総合分析レポート**を自動生成する **非公式 Claude Skill** です。

Claude Code / Claude Desktop の両方で動作します。

## 何ができるか

### (A) 総合分析レポート
- ビジネスモデルの把握（主要取引先・仕入先・回収サイト・支払サイト）
- 定期取引パターンの自動抽出
- 給与・人件費の月次トレンド
- 部門別分析（部門運用している事業者）
- 入力元分析（データ連携・給与連携・手入力の内訳）

### (B) 入力担当者向け引き継ぎシート（Step 1〜9 単一MDファイル）
- **Step 4**: BS科目×補助×部門×発生/消込 の入力ソース一覧（引き継ぎ書の骨格）
- **Step 5**: 月次作業フロー（フェーズ1〜6 排他構造で二重記帳防止）
- **Step 9**: 通帳カタカナ → 仕訳科目の逆引きルールブック（残高照合画面から自動集約）
- 登録経路16カテゴリで仕訳を機械分類、Step 5 網羅性検証で漏れを自動検出

## 特徴

- **BS科目×補助科目×部門×発生/消込** の粒度で入力ソースを管理（二重記帳の構造的防止）
- **1ターン1科目** の対話型設計（認知負荷を下げる）
- **登録経路16カテゴリ**（データ連携自動取得/手動管理/MF給与連携/CSVインポート/MF経費連携/MF請求書連携/MF債務支払連携/STREAMED連携/モバイルアプリ/AI-OCR/デジタルインボイス/外部API連携/家事按分/償却仕訳/開始残高/手入力）で機械判定
- **連携サービス名と補助科目のマッチング率検証**（Phase 2-1d）で補助科目リネーム時の判定漏れを防止
- 残高照合画面テキストをパースして **逆引きルールブック** を生成（JSON突合で実仕訳の科目・補助・税区分を正確に記載）

## セットアップ

### 前提: MF公式MCPサーバーの登録

このスキルは MF公式 MCP（リモートMCPコネクタ）を使います。**別途登録が必要**です。

[mf-official-mcp-skill](https://github.com/kentaroajisaka/unofficial-official-mf-mcp-skill) の手順に従って MF公式 MCPを登録してください。

### Claude Desktop の場合

1. [Releases](../../releases) からzipファイルをダウンロード
2. Claude Desktop を開く
3. 左サイドバーの「カスタマイズ」→「スキル」→「+」
4. ダウンロードしたzipを選択

### Claude Code の場合

```bash
# スキルフォルダに配置
mkdir -p ~/.claude/skills/mfc-journal-analyst-pro
cd ~/.claude/skills/mfc-journal-analyst-pro

# GitHubから取得
curl -L https://github.com/kentaroajisaka/mfc-journal-analyst-pro/archive/refs/tags/v1.0.0.tar.gz | tar xz --strip-components=1
```

### 依存ライブラリ

- Python 3.9+
- pandas（スキル起動時に自動インストール案内あり）

## 使い方

### 基本フロー

Claude Code / Claude Desktop で以下のようにリクエストすると自動発動します：

- 「〇〇社のMFの仕訳を分析したい」
- 「〇〇社の引き継ぎ書を作成して」
- 「記帳代行の引き継ぎ資料を作って」

### 実行フロー（Phase 4B の場合）

1. **認証**: MF公式MCP（beta版は自動・alpha版は手動コピペ）
2. **事業者情報・連携サービス・部門マスタ取得**（並列API呼出）
3. **登録済み一覧のスクリーンショット依頼**（手動管理・自動取得の全サービス把握）
4. **補助科目とのマッチング率検証**（リネーム疑いの検出）
5. **仕訳取得** + 経理方式判定（getTermSettings で一発）
6. **BS科目×補助×部門 × 発生/消込 の洗い出し** + 入力ソース推定
7. **1ターン1科目対話**で入力ソース・証憑を確定（推定が合っていれば「OK」だけ）
8. **登録経路16カテゴリの排他性チェック**（二重記帳の構造的防止）
9. **Step 5 網羅性検証**（漏れパターンの自動検出）
10. **引き継ぎ書 Step 1〜9 生成**（単一MDファイル、残高照合テキストがあれば Step 9 逆引きマップも自動生成）

## 想定ユーザー

- 会計事務所の税理士・スタッフ（記帳代行の引き継ぎ資料を作りたい）
- 顧問先の経理責任者（属人化した記帳フローを標準化したい）
- MFクラウド会計で複数法人を扱う担当者

## 注意事項

- これは **非公式** のスキルです。マネーフォワード社が提供・サポートするものではありません
- MF公式 MCPサーバー（リモートMCPコネクタ）は別途設定が必要です
- 生成された引き継ぎ書は**推論ベース**です。本番使用前にユーザーとの対話で各科目の登録経路・証憑を確定することを推奨します
